=====================================================
 CLOUDFLARE TUNNEL CONFIGURATION
 Exposing LMS via public domain
=====================================================

Since you already have a Cloudflare Tunnel running for other
purposes, you just need to add a new public hostname.

STEP 1: Check existing tunnel
-------------------------------
cloudflared tunnel list

Note your tunnel name or ID.

STEP 2: Add LMS hostname to tunnel config
-------------------------------------------
Edit your tunnel config file:
  nano ~/.cloudflared/config.yml

Add a new ingress rule for the LMS. Your config should look
something like this:

----------- CONFIG -----------

tunnel: YOUR_TUNNEL_ID
credentials-file: /home/YOUR_USER/.cloudflared/YOUR_TUNNEL_ID.json

ingress:
  # Your existing services
  - hostname: existing.e2eindia.org
    service: http://localhost:EXISTING_PORT

  # ADD THIS for the LMS
  - hostname: lms.e2eindia.org
    service: http://localhost:80

  # Catch-all (must be last)
  - service: http_status:404

----------- END CONFIG -----------


STEP 3: Add DNS record in Cloudflare
--------------------------------------
Option A - Via CLI:
  cloudflared tunnel route dns YOUR_TUNNEL_NAME lms.e2eindia.org

Option B - Via Cloudflare Dashboard:
  1. Go to dash.cloudflare.com
  2. Select your domain (e2eindia.org)
  3. Go to DNS
  4. Add a CNAME record:
     Name: lms
     Target: YOUR_TUNNEL_ID.cfargotunnel.com
     Proxy: ON (orange cloud)

STEP 4: Restart the tunnel
----------------------------
sudo systemctl restart cloudflared

OR if running manually:
  cloudflared tunnel run YOUR_TUNNEL_NAME

STEP 5: Verify
----------------
Wait 1-2 minutes for DNS propagation, then:
  curl https://lms.e2eindia.org
  curl https://lms.e2eindia.org/api/health

Both should return valid responses.

STEP 6: Update frontend API URL
---------------------------------
Make sure the frontend .env.production has the correct URL:
  cd /var/www/lms/frontend
  echo 'VITE_API_URL=https://lms.e2eindia.org/api' > .env.production
  npm run build
  sudo cp -r dist/* /var/www/lms/public/


NOTES
------
- Cloudflare provides free SSL, so no need for certbot
- The tunnel handles the double-NAT bypass automatically
- Max upload size: Cloudflare free plan allows 100MB uploads
  through the tunnel. For larger video files, you may need to:
  a) Upload directly on the server (via SSH/SCP)
  b) Use Cloudflare's Enterprise plan
  c) Upload through a direct IP connection on your local network
- For large class videos (200MB+), recommend uploading via
  the local network: http://SERVER_LOCAL_IP/admin
